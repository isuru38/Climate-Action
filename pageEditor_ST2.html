Techniques & Explanation

1. Separated Page-specific CSS

Created a dedicated CSS files to improve modularity and performance. This ensures clean separation between global styles and editor-specific design.

Benefits:

  • Improved maintainability through modular code structure
  • Faster load time for unrelated pages (avoids unused CSS)
  • Allows easier debugging and future scalability

2. Global CSS and Website Style

Developed a global stylesheet style.css for consistent branding and layout across all general pages. This includes typography, color schemes, header and footer design, and responsive navigation.

Key Techniques:

  • CSS Variables for theming and easy color management
  • Flexbox used to create flexible header and navigation layouts
  • Media queries for adapting to different screen sizes
  • Smooth transition effects for interactive UI elements like links and buttons

3. Home Page

The homepage features a welcoming mission statement and interactive card links to content sections.

Key Features:

  • CSS Grid layout for arranging the interactive cards
  • Hover effects using opacity and transform for dynamic user feedback
  • Absolute positioning for text overlays on cards
  • Responsive design with flexible card wrapping on smaller screens

4. Table Page

Created an interactive climate data table with clear visual hierarchy and accessibility considerations.

Technical Implementation:

Feature Implementation
Merged Cells Used rowspan and colspan attributes for grouping related data
Zebra Striping Applied :nth-child(even) and :nth-child(odd) CSS selectors
Hover Effects Used transform: scale(1.01) and background color changes on row hover
Status Indicators Color-coded text and icons using CSS classes for quick status recognition
Accessible Table Markup Used scope attributes and proper semantic tags for screen readers

Accessibility

Implemented accessibility features across all pages:

  • Semantic HTML5 structure (header, nav, main, footer)
  • Alt text for all images
  • Keyboard-navigable interface
  • Color contrast ratio > 4.5:1 for readability
  • ARIA labels where needed

Challenges & Lessons Learned

Key challenges faced during development:

  • Responsive Tables: Making the data table readable on mobile required careful media query implementation
  • Image Paths: Ensuring image paths worked correctly across different devices
  • CSS Organization: Maintaining clean and reusable CSS across multiple pages

Key Lessons:

  • Importance of mobile-first development approach
  • Benefits of semantic HTML for accessibility
  • Value of consistent file organization
  • Effective use of CSS Grid and Flexbox
