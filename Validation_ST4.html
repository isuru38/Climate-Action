Web Standards Validation

Ensuring our climate action platform meets all web accessibility and quality standards

1. Validation Process

Tool Selection & Setup

All pages were validated using the W3C Markup Validator as specified in the "Required Software & Access to Services" section on Blackboard. The validator ensures compliance with HTML5 standards and web accessibility guidelines.

Comprehensive Page Testing

Each implemented page was systematically tested through the validation tool. All errors were identified, documented, and resolved to ensure full compliance with web standards.

Error Resolution

All validation errors have been successfully resolved. Warnings were reviewed and addressed where applicable to ensure optimal accessibility and user experience.

4. Reflection

Validation Process Reflection

Process Effectiveness: The systematic validation process using W3C standards proved highly effective in identifying and resolving web accessibility and compliance issues. Each page was carefully reviewed and optimized to meet current web standards.

Key Improvements Made: During validation, we enhanced semantic HTML structure, improved accessibility features including alt text and ARIA labels, optimized form structures, and ensured proper heading hierarchy across all pages.

Standards Compliance: All pages now fully comply with HTML5 standards and WCAG accessibility guidelines, ensuring our climate action platform is accessible to all users regardless of their abilities or the devices they use.

Future Considerations: Regular validation checks will be implemented as part of our ongoing development process to maintain high standards of web accessibility and user experience as we continue to expand our climate action resources.
